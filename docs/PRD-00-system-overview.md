## 1. 프로젝트 개요

### 1.1 프로젝트명
**부동산 경매 분석 AI 시스템** (Real Estate Auction Analysis AI System)

### 1.2 프로젝트 목적
사용자가 부동산 법원 경매 정보를 제공하면, AI 에이전트가 경매 물건을 종합적으로 분석하여 **입찰 여부 추천**, **입찰 적정가**, **매도 적정가**를 산출해주는 시스템을 구축한다.

### 1.3 핵심 가치 제안
- 경매 초보자도 전문가 수준의 분석 결과를 얻을 수 있다
- 권리분석, 시세분석, 시장동향을 종합한 객관적 판단 제공
- 수익률 기반의 투자 의사결정 지원

## 3. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| **Frontend** | React + TypeScript | SPA, 반응형 웹 (Node.js 20.19+ 또는 22.12+) |
| **Backend** | FastAPI (Python) | 비동기 API, uv venv 가상환경 |
| **AI Agent** | LangGraph + DeepAgent | 멀티 에이전트 워크플로우 |
| **LLM** | Anthropic Claude (Sonnet/Opus) | 경매 분석 및 자연어 처리 |
| **PDF 파싱** | PyPDF2 / pdfplumber / Unstructured | 경매 문서 텍스트 추출 |
| **데이터베이스** | SQLite / PostgreSQL | 분석 결과 및 이력 저장 |
| **실거래가 API** | 국토교통부 공공데이터 API | 실거래가 조회 |
| **패키지 관리** | uv | Python 패키지 및 가상환경 관리 |

## 4. 기능 모듈 구성

### Module 1: 경매 문서 파싱 (PRD-01)
- PDF 업로드 및 텍스트 추출
- 문서 유형 자동 분류 (등기부등본, 매각물건명세서, 감정평가서 등)
- 구조화된 데이터 추출

### Module 2: 권리분석 에이전트 (PRD-02)
- 등기부등본 기반 권리관계 분석
- 인수되는 권리 / 소멸되는 권리 판별
- 위험요소 식별 및 등급 산정

### Module 3: 시장 데이터 수집 (PRD-03)
- 국토교통부 실거래가 API 연동
- 주변 유사 매물 시세 조회
- 지역별 시세 트렌드 분석

### Module 4: 뉴스 및 시장 동향 분석 (PRD-04)
- 해당 지역 관련 뉴스 수집
- 개발 호재/악재 분석
- 부동산 시장 전반 트렌드 분석

### Module 5: 가치 평가 및 적정가 산출 (PRD-05)
- 입찰 적정가 산출 알고리즘
- 매도 적정가 산출 알고리즘
- 투자 수익률 분석
- 최종 입찰 추천 여부 판단

### Module 6: AI 에이전트 시스템 (PRD-06)
- LangGraph 기반 워크플로우 설계
- DeepAgent 통합 전략
- 에이전트 간 데이터 흐름 관리
- 오류 처리 및 재시도 전략

### Module 7: 백엔드 API (PRD-07)
- FastAPI 기반 REST API 설계
- 파일 업로드 및 관리
- 분석 작업 관리 (비동기)
- 결과 조회 및 이력 관리

### Module 8: 프론트엔드 (PRD-08)
- React + TypeScript SPA
- PDF 업로드 인터페이스
- 분석 진행 상태 표시
- 결과 대시보드 및 리포트 뷰어

## 5. 사용자 시나리오 (User Flow)

```
1. 사용자가 웹 페이지에 접속
       │
2. 경매 관련 PDF 문서 업로드
   (등기부등본, 매각물건명세서, 감정평가서 등)
       │
3. 시스템이 문서를 파싱하여 경매 정보 추출
       │
4. AI 에이전트가 병렬로 분석 수행
   ├─ 권리분석 (등기부등본 기반)
   ├─ 시장 데이터 수집 (실거래가, 시세)
   └─ 뉴스 및 시장 동향 분석
       │
5. 종합 분석 결과를 기반으로 가치 평가
   ├─ 입찰 적정가 산출
   ├─ 매도 적정가 산출
   └─ 투자 수익률 계산
       │
6. 최종 분석 리포트 제공
   ├─ 입찰 추천 여부 (추천/보류/비추천)
   ├─ 입찰 적정가 범위
   ├─ 예상 매도가 범위
   ├─ 예상 수익률
   ├─ 위험요소 요약
   └─ 상세 분석 근거
```

## 6. 비기능 요구사항

### 6.1 성능
- PDF 파싱: 단일 문서 기준 10초 이내
- 전체 분석 프로세스: 5분 이내 완료
- 동시 사용자: 최소 10명 동시 분석 지원

### 6.2 보안
- 업로드된 PDF 파일 암호화 저장
- 분석 완료 후 일정 기간 경과 시 파일 자동 삭제
- API 키 및 민감 정보 환경변수 관리

### 6.3 사용성
- 직관적인 UI (경매 초보자도 쉽게 사용 가능)
- 분석 결과를 시각적으로 이해하기 쉽게 제공
- 모바일 반응형 지원

### 6.4 확장성
- 새로운 분석 에이전트 추가 용이한 구조
- LLM 모델 교체 가능한 추상화 레이어
- 추후 경매 물건 DB 구축 가능한 구조

## 9. 용어 정의

| 용어 | 설명 |
|------|------|
| **경매** | 법원이 채무자의 부동산을 강제로 매각하는 절차 |
| **감정가** | 법원이 선임한 감정인이 평가한 부동산의 가치 |
| **최저매각가격** | 입찰할 수 있는 최소 금액 (회차마다 20% 감소) |
| **등기부등본** | 부동산의 소유권, 권리관계가 기재된 공적 문서 |
| **매각물건명세서** | 경매 대상 물건의 상세 정보를 기재한 법원 문서 |
| **감정평가서** | 감정인이 작성한 부동산 가치 평가 보고서 |
| **권리분석** | 등기부등본 등을 분석하여 권리관계를 파악하는 작업 |
| **인수** | 낙찰자가 떠안아야 하는 기존 권리 |
| **소멸** | 경매로 인해 없어지는 기존 권리 |
| **실거래가** | 실제 거래된 가격 (국토교통부 신고 기준) |

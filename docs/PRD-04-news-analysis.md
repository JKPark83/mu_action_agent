## 1. 개요

### 1.1 목적
경매 대상 부동산이 위치한 지역의 뉴스와 시장 동향을 수집하고 분석하여, 해당 지역의 개발 호재/악재 및 부동산 시장 전망을 파악하는 에이전트를 개발한다.

### 1.2 배경
부동산 가치는 주변 개발 계획, 교통 인프라 변화, 정부 정책 등 외부 요인에 크게 영향을 받는다. 이러한 정보를 체계적으로 수집하고 분석하면 보다 정확한 가치 평가가 가능하다.

## 2. 데이터 수집 영역

### 2.1 지역 뉴스
| 분류 | 키워드 예시 | 영향 |
|------|------------|------|
| **개발 호재** | 신도시, 재개발, 재건축, GTX, 지하철 연장 | 가격 상승 요인 |
| **개발 악재** | 혐오시설, 공장 이전, 인구 감소 | 가격 하락 요인 |
| **교통** | 도로 신설, 역세권 개발, 버스 노선 변경 | 접근성 변화 |
| **교육** | 학군 변경, 학교 신설, 특목고 설립 | 주거 수요 영향 |
| **생활 인프라** | 대형마트, 병원, 공원 조성 | 생활 편의성 변화 |

### 2.2 부동산 정책
| 분류 | 내용 |
|------|------|
| **세금 정책** | 취득세, 양도세, 종부세 변경 |
| **대출 규제** | LTV, DTI, DSR 변경 |
| **공급 정책** | 분양 물량, 임대주택 공급 |
| **투기과열지구** | 지정/해제 여부 |
| **규제 지역** | 조정대상지역 지정/해제 |

### 2.3 시장 동향
| 지표 | 설명 |
|------|------|
| **매매가격지수** | KB, 한국부동산원 기준 |
| **전세가격지수** | 전세 시장 동향 |
| **미분양 현황** | 해당 지역 미분양 물량 |
| **인구 변동** | 전입/전출 추이 |
| **경매 낙찰률** | 해당 지역 경매 시장 활성도 |

## 3. 기능 요구사항

### 3.1 뉴스 수집
- **FR-04-001**: 경매 물건 소재지 기반으로 관련 뉴스를 자동 수집해야 한다
- **FR-04-002**: 최근 6개월~1년 이내의 뉴스를 수집 범위로 설정한다
- **FR-04-003**: 뉴스의 신뢰도(출처)를 기반으로 가중치를 부여해야 한다
- **FR-04-004**: 중복 뉴스를 제거하고 핵심 뉴스만 추출해야 한다

### 3.2 뉴스 분석
- **FR-04-005**: 수집된 뉴스를 호재/악재/중립으로 분류해야 한다
- **FR-04-006**: 각 뉴스의 부동산 가치 영향도를 점수화해야 한다
- **FR-04-007**: 뉴스 내용을 요약하여 핵심 정보만 추출해야 한다
- **FR-04-008**: 시간 경과에 따른 영향도 감쇠를 반영해야 한다

### 3.3 시장 동향 분석
- **FR-04-009**: 해당 지역의 부동산 시장 전반적인 동향을 분석해야 한다
- **FR-04-010**: 향후 6개월~1년의 시장 전망을 예측해야 한다
- **FR-04-011**: 전국 평균 대비 해당 지역의 시장 상황을 비교 분석해야 한다

### 3.4 종합 의견 생성
- **FR-04-012**: 수집된 뉴스와 동향을 종합한 지역 분석 의견을 생성해야 한다
- **FR-04-013**: 투자 관점에서의 지역 매력도를 점수화해야 한다
- **FR-04-014**: 주요 호재/악재를 우선순위별로 정리해야 한다

## 5. 데이터 모델

### 5.1 뉴스 분석 결과
```python
@dataclass
class NewsAnalysisResult:
    """뉴스 분석 결과"""
    target_area: str                    # 분석 대상 지역
    analysis_date: date                 # 분석 시점

    # 수집된 뉴스
    collected_news: list[AnalyzedNews]

    # 호재/악재 요약
    positive_factors: list[Factor]      # 호재 목록
    negative_factors: list[Factor]      # 악재 목록

    # 시장 동향
    market_trend: MarketTrendSummary

    # 종합 의견
    area_attractiveness_score: float    # 지역 매력도 (0~100)
    investment_opinion: str             # 투자 관점 종합 의견
    outlook: str                        # "긍정" | "중립" | "부정"

    confidence_score: float             # 분석 신뢰도

@dataclass
class AnalyzedNews:
    """분석된 개별 뉴스"""
    title: str                          # 뉴스 제목
    source: str                         # 출처
    published_date: date                # 게시일
    url: str                            # 원문 URL
    summary: str                        # 요약
    sentiment: str                      # "호재" | "악재" | "중립"
    impact_score: float                 # 영향도 (0~10)
    category: str                       # 분류 (개발, 교통, 교육 등)

@dataclass
class Factor:
    """호재/악재 요소"""
    title: str                          # 요소명
    description: str                    # 상세 설명
    impact_level: str                   # "높음" | "중간" | "낮음"
    timeline: str                       # 영향 시기 (단기/중기/장기)
    source_news: list[str]              # 근거 뉴스 제목

@dataclass
class MarketTrendSummary:
    """시장 동향 요약"""
    price_direction: str                # "상승" | "하락" | "보합"
    transaction_volume: str             # "증가" | "감소" | "유지"
    supply_demand: str                  # "공급과잉" | "수요초과" | "균형"
    policy_impact: str                  # 정책 영향 요약
    forecast_6m: str                    # 6개월 전망
```

## 6. AI 에이전트 설계

### 6.1 Claude 활용 포인트
- **키워드 생성**: 소재지 기반 검색 키워드 자동 생성
- **뉴스 분류**: 호재/악재/중립 자동 분류
- **요약**: 뉴스 내용 핵심 요약
- **종합 분석**: 다수의 뉴스와 데이터를 종합한 투자 의견 생성
- **전망 생성**: 수집된 정보 기반 시장 전망 생성

### 6.2 분석 파이프라인
```
1. 소재지 정보에서 검색 키워드 생성
   (예: "강남구 역삼동 재개발", "역삼역 개발 호재")
       │
2. 뉴스 API를 통해 관련 뉴스 수집
       │
3. Claude를 활용하여 뉴스 분류 및 요약
       │
4. 시장 데이터 API에서 동향 지표 수집
       │
5. 전체 데이터를 종합하여 지역 분석 리포트 생성
```

## 7. 비기능 요구사항

- **수집 건수**: 지역당 최소 20건, 최대 50건의 관련 뉴스 수집
- **분석 시간**: 뉴스 수집 및 분석 완료 120초 이내
- **API 제한**: 뉴스 API rate limit 준수
- **데이터 유효기간**: 뉴스 분석 결과는 7일 간 캐싱

## 1. 개요

### 1.1 목적
사용자가 경매 문서를 업로드하고 분석 결과를 확인할 수 있는 직관적인 웹 인터페이스를 React + TypeScript로 구현한다.

### 1.2 기술 스택
| 항목 | 기술 | 비고 |
|------|------|------|
| **런타임** | Node.js 20.19+ 또는 22.12+ | Vite 7 요구사항 |
| **프레임워크** | React 19+ | SPA (Single Page Application) |
| **언어** | TypeScript | 타입 안전성 |
| **빌드 도구** | Vite | 빠른 빌드 및 HMR |
| **상태 관리** | TanStack Query (React Query) | 서버 상태 관리 |
| **UI 라이브러리** | Tailwind CSS + shadcn/ui | 유틸리티 기반 스타일링 |
| **차트** | Recharts | 시세 차트 시각화 |
| **HTTP 클라이언트** | Axios / fetch | API 호출 |
| **WebSocket** | 기본 WebSocket API | 실시간 상태 업데이트 |

## 4. 기능 요구사항

### 4.1 파일 업로드
- **FR-08-001**: 드래그 앤 드롭으로 PDF 파일을 업로드할 수 있어야 한다
- **FR-08-002**: 파일 선택 다이얼로그를 통해서도 업로드할 수 있어야 한다
- **FR-08-003**: 업로드 진행률을 표시해야 한다
- **FR-08-004**: 잘못된 파일 형식 업로드 시 에러 메시지를 표시해야 한다
- **FR-08-005**: 업로드된 파일을 목록에서 확인하고 삭제할 수 있어야 한다

### 4.2 분석 진행 상태
- **FR-08-006**: WebSocket을 통해 실시간 진행 상태를 표시해야 한다
- **FR-08-007**: 각 분석 단계별 진행률을 시각적으로 표시해야 한다
- **FR-08-008**: 현재 진행 중인 작업에 대한 설명을 표시해야 한다
- **FR-08-009**: 예상 완료 시간을 표시해야 한다
- **FR-08-010**: 에러 발생 시 사용자에게 알림을 표시해야 한다

### 4.3 분석 결과 리포트
- **FR-08-011**: 종합 판단 결과(추천/보류/비추천)를 시각적으로 명확히 표시해야 한다
- **FR-08-012**: 입찰 적정가, 매도 적정가, 수익률을 표시해야 한다
- **FR-08-013**: 시세 변동 추이를 차트로 시각화해야 한다
- **FR-08-014**: 권리분석, 시세분석, 뉴스/동향, 비용 분석을 탭으로 분리해야 한다
- **FR-08-015**: 면책 조항을 리포트 하단에 표시해야 한다
- **FR-08-016**: 분석 결과를 PDF로 다운로드할 수 있어야 한다 (향후)

### 4.4 분석 이력
- **FR-08-017**: 과거 분석 작업 목록을 조회할 수 있어야 한다
- **FR-08-018**: 과거 분석 결과를 다시 볼 수 있어야 한다

### 4.5 UI/UX
- **FR-08-019**: 모바일 반응형 디자인을 적용해야 한다
- **FR-08-020**: 로딩 상태에 적절한 스켈레톤 UI를 표시해야 한다
- **FR-08-021**: 경매 초보자도 이해할 수 있는 직관적인 UI를 제공해야 한다

## 6. 주요 타입 정의

```typescript
// 분석 상태
type AnalysisStatus = 'pending' | 'running' | 'done' | 'error';

// 추천 결과
type Recommendation = '추천' | '보류' | '비추천';

// 분석 진행 상태
interface AnalysisProgress {
  overall: number;
  stages: {
    document_parsing: StageStatus;
    rights_analysis: StageStatus;
    market_data: StageStatus;
    news_analysis: StageStatus;
    valuation: StageStatus;
    report_generation: StageStatus;
  };
}

interface StageStatus {
  status: AnalysisStatus;
  progress: number;
}

// 분석 결과 리포트
interface AnalysisReport {
  id: string;
  case_number: string;
  property_address: string;
  recommendation: Recommendation;
  recommendation_reason: string;
  bid_price: PriceRange;
  sale_price: PriceRange;
  profitability: ProfitabilityAnalysis;
  cost_breakdown: CostBreakdown;
  risk_assessment: RiskAssessment;
  confidence_score: number;
  disclaimer: string;
  created_at: string;
}

interface PriceRange {
  conservative: number;
  moderate: number;
  aggressive: number;
  minimum_bid?: number;
}
```
